El **modelo de Von Neumann** es una arquitectura de computadoras que propone que: - Los datos y las instrucciones se almacenan en la misma memoria. - La CPU procesa las instrucciones siguiendo el ciclo **fetch → decode → execute**. - El acceso a memoria se realiza mediante un bus compartido. Este modelo es la base de la mayoría de los ordenadores actuales. --- # docs/seccion1/sub1_2.mds y limitaciones. - Comparaciones con otros modelos. --- # docs/seccion1/sub1_1.md # Sub 1.1 Contenido de la sección 1.1. --- # docs/seccion1/sub1_2.md # Sub 1.2 – Componentes principales La arqui# docs/seccion2.md # Sección 2 – Comparación con Harvard La **arquitectura Harvard** se diferencia de la de Von Neumann en que: - Separa la memoria de datos y la memoria de instrucciones. - Permite acceder a ambas memorias simultáneamente. - Evita el llamado _cuello de Von Neumann_. ## Ventajas de Harvard - Mayor rendimiento en ciertos sistemas embebidos. - Menor riesgo de errores al separar instrucciones y datos. ## ¿Por qué se usa más Von Neumann? - Es más simple y barata de implementar. - Adecuada para la mayoría de computadoras de propósito general. rocesamiento (CPU) Encargada de ejecutar instrucciones y realizar cálculos. ## 2.
